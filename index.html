News Mobile App Project Report

A complete summary of the features, design, and technical details of the News Mobile App.

Joel's Approach: Scientific Design

Start with Research.

Before I started building the News Mobile App, I asked important questions like: "What do users want in a news app?" and "How can I make it simple and helpful?"

I found out that users need:

  • News that is updated daily so they always see the latest stories.
  • An offline feature to save articles for when they don’t have an internet connection.
  • A search bar so they can quickly find news about topics they are interested in.

These needs helped me decide what features to include. For example, the app saves articles locally, so even if there’s no internet, users can still read the news they saved. The search feature also makes it easy for users to look for specific topics.

By asking good questions and focusing on what users need, I was able to create a helpful and user-friendly news app.

Joel's Approach : Decision Analysis

Evaluating Ideas Effectively

During the planning phase of the News Mobile App, I carefully evaluated ideas to ensure they aligned with user needs and project goals. I focused on practical solutions while avoiding ideas that sounded good but didn’t serve a real purpose.

For example, I initially considered adding complex animations to make the app visually appealing. However, after analyzing the decision, I realized that users value fast loading times and easy navigation more than animations. So, I chose a clean and responsive design instead.

This approach of evaluating and eliminating unnecessary features saved time and improved the app’s user experience. By focusing on what matters most, I ensured the app met its goals effectively.

Key Features

  • Headlines: Real-time updates of the latest news.
  • Favorites: Save and manage articles offline.
  • Search: Find news articles using keywords.

Project Overview

The News Mobile App aggregates real-time news using APIs and displays them in an easy-to-navigate interface. It integrates modern concepts like MVVM architecture, Room Database, Retrofit for API calls, and Coroutines for background tasks.

  • MVVM Architecture for modularity and clear separation of concerns.
  • Room Database for offline storage of favorite articles.
  • Navigation Component for intuitive screen transitions.
  • Retrofit for network calls and API integration.

Technical Details

  • API: Integrated with News API for fetching news data.
  • Room Database: For storing saved articles locally on the device.
  • Coroutines: Efficient background threading for smooth UI operations.
  • Navigation Component: Simplified navigation and backstack handling.

RSS Integration in the News Mobile App

Technologies and Tools Used

  • Volley / Picasso: Used for efficient network requests and image loading, ensuring fast and smooth performance.
  • SAX or DOM: Implemented for parsing XML data from RSS feeds, making the integration of news sources seamless.
  • Database Access: ROOM database is used for local storage of articles, allowing users to save and view content offline.
  • Static Config File: Centralized file in the util package to store reusable constants, such as API endpoints and keys.
  • Caffeine Library: Used for caching frequently accessed data, reducing load times and improving the user experience.

How These Fit in the Project Structure

The RSS integration is well-organized within the project structure:

  • api/NewsAPI and api/RetrofitInstance: Handle network requests for fetching news data and images.
  • db/ArticleDatabase and db/Converters: Store articles locally using ROOM for offline access.
  • repository/NewsRepository: Serves as the bridge between the database and the UI, ensuring seamless data flow.
  • ui/fragments: Displays parsed RSS feed data in the app through fragments like HeadlinesFragment and FavouritesFragment.
  • util/Constants: Contains static configuration for managing API URLs and other shared resources.

Benefits of the Implementation

This structure ensures that the app remains:

  • Fast and responsive due to caching with Caffeine and optimized network requests.
  • Reliable even without an internet connection, thanks to ROOM database integration.
  • Scalable and maintainable with a clean separation of concerns between components.

User Flow

The app provides a seamless navigation experience through the following screens:

  1. Headlines: Display the latest articles.
  2. Favorites: Show saved articles.
  3. Search: Let users search for news using keywords.

Offline Support in the News Mobile App

What Happens When Users Have No Internet?

The News Mobile App is designed to work seamlessly even when users have no internet connection. This ensures users can still access saved articles and enjoy the app without interruptions.

Key Features for Offline Support

  • ROOM Database: The app stores articles locally so users can save their favorite articles and read them offline anytime.
  • Accessing Saved Content: Users can navigate to the Favourites section to view all saved articles.
  • Error Handling: When the app detects no internet connection, it shows a message encouraging users to check their saved content in Favourites.

How It Works

The NewsRepository determines the source of the data. When no internet is available, it retrieves data from the local database instead of making API calls. FavouritesFragment and HeadlinesFragment are updated dynamically to display this content.

Critical Analysis

Challenges and Areas for Improvement

  • Favourites Offline Loading: When the user is offline, the Favourites section does not always load saved articles correctly. This can cause frustration, as users rely on this feature to access content without an internet connection.
  • Search Functionality: The search feature does not work offline, as it requires an API call to fetch results. Adding a fallback to search locally stored articles could improve usability.
  • Error Handling: When there is no internet connection, error messages could be clearer and more user-friendly, suggesting alternative actions like navigating to Favourites or retrying later.
  • UI Responsiveness: On certain devices, the layout of fragments like HeadlinesFragment becomes misaligned in landscape mode. Improved testing and adjustments to the layout could fix this issue.
  • Performance: While the app is functional, large volumes of saved articles can slow down the loading of the Favourites section. Optimizing database queries and implementing pagination could address this.

Conclusion

Working on the News Mobile App has been a great learning experience. I learned many important skills and concepts during this project.

  • API Integration: I now understand how to connect to APIs and fetch real-time data, which made the app dynamic and useful.
  • Understanding DAO: I learned about Data Access Objects (DAO) and how they help in managing the database to store and retrieve articles efficiently.
  • Joel Marsh Principles: Following Joel Marsh’s principles, I realized the importance of research and decision analysis. These helped me create a better app by focusing on what users really need.
